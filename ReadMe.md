Принципы __SOLID__
1. Single Responsibility Principle (Принцип единственной ответственности).
Считаю, что данный принцип полностью выполнен в данном приложении
2. Open Closed Principle (Принцип открытости/закрытости).
В классе User если есть необходимость создать новое поле (например адрес), то возникнут сложности. Я так думаю можно как-то создать абстракный класс, от которого все поля User будут наследоваться. И отдельно списком задавать имена полей и их тип. Если надо добавить новые данные (email, заметки и т.д., то в список добавлять, а не вносить изменения в класс) 
3. Liskov’s Substitution Principle (Принцип подстановки Барбары Лисков).
Этот принцип необходимо будет реализовать, если учесть, что написано в п.2
